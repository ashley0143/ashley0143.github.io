Privacy Settings

Use the toggle below to instantly enable or disable metrics collection. No site reload required.

Metrics Collected & Purpose
  • Page Load Timings: Benchmark resource loading and optimize asset prioritization.
  • Click Events: Identify interaction hotspots and refine UI/UX elements.
  • Form Submissions: Streamline form processes and error handling flows.
  • Copy & Paste Actions: Gauge content engagement and user workflows.
  • Scroll Depth Measurements: Determine content visibility and adjust layout accordingly.
  • Visibility & Focus Changes: Detect tab/window usage to manage background tasks.
  • Orientation Events: Adapt layouts to device orientation changes.
  • Device Memory & JS Heap Stats: Monitor client constraints to prevent performance degradation.
  • Heartbeat & Idle Tracking: Distinguish active usage from idle periods for session analytics.
  • JavaScript Error Logging: Capture runtime errors to prioritize bug fixes.
Privacy Policy

TL;DR: On-device metrics are collected solely locally for performance & UX improvements; data never leaves your browser. You may disable collection or clear data at any time.

Effective Date

This policy is effective as of May 19, 2025, and supersedes all prior versions.

1. Introduction & Acceptance

By accessing or using this website, you acknowledge that you have read, understood, and agree to be bound by this Privacy Policy and Terms of Service. If you do not agree with any part of these terms, you must cease usage immediately.

2. Hosting & Third-Party Resources

This website is hosted on GitHub Pages. Third-party domains (e.g., font CDNs, image hosts) are exclusively used for serving static assets. These providers are contractually restricted from collecting any usage data or metrics related to this site.

3. Data Collection Scope

We collect non-identifiable, aggregated metrics locally in your browser’s localStorage. No data is ever transmitted to external servers, ensuring that all analytics remain on-device.

4. Data Types & Purpose

The following anonymous data are collected with their respective purposes:

  • Performance Metrics (e.g., load times): To enhance page speed and reduce latency.
  • Interaction Events (clicks, submissions): To optimize UI placement and workflows.
  • Scroll & Visibility Data: To adjust content hierarchy and manage background tasks.
  • Error Logs: To identify and fix JavaScript defects promptly.

5. Data Minimization & Anonymity

Adhering to data minimization principles, we only collect metadata necessary for improving the site. No personal identifiers, IP addresses, or geolocation data are ever recorded.

6. Data Security & Confidentiality

Data stored in localStorage is sandboxed within your browser and domain context. It cannot be accessed by other origins or external scripts not served by this site.

7. Retention Period

Metrics persist indefinitely until you delete them. There is no automatic expiration to allow for long-term trend analysis if desired.

8. User Rights & Controls

You have the right to control and manage your analytics data at any time through the following mechanisms:

  • _hs.get() — Retrieve all stored metrics.
  • _hs.export() — Download metrics as a JSON file.
  • _hs.clear() — Permanently delete all metrics from localStorage.
  • Toggle Collection — Instantly enable or disable data collection via the switch above.

9. Privacy-by-Design & Regulatory Alignment

This policy reflects privacy-by-design practices. Although no PII is collected, we align with GDPR and CCPA principles by ensuring data minimization, transparency, and user consent mechanisms.
